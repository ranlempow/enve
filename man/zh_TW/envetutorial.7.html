NAME

envetutorial - 首次介紹 enve

SYNOPSIS

enve *

DESCRIPTION

enve是一系列的命令工具用來協助開發者們共同作業，這包含維持一致的開發環境、分享團隊中的資源與工具、管理團隊中的服務與工作。

ABILITY 功能

  1. 環境建置自動化
  2. 原始碼協作與管理
  3. 部署自動化
  4. 持續整合與持續發行
  5. 權限與機密控管
  6. 專案團隊管理

FEATURE 特色

易用
命令列的用法非常直觀而且一體通用，報錯訊息淺顯易懂，文件豐富概念單純。
簡單
全部程式碼以古老的SHELL語法寫成，只有少數如openssl等選擇性的外部依賴，不使用任何複雜演算法，即使用其他語言重新實作也非常容易。
模組化
在無任何前提下不用編譯即可相容任何POSIX系統。透過公開穩定的協定與其他軟體合作，你不一定要使用enve的全部部分，而且可將任何enve組件替換成其他的軟體，或是將enve結合到其他軟體。
去中央
所有的操作都可以在本地完成，不須依賴中央伺服器。

WHY ENVE 為何要用

無論是個人或小型團隊都能利用enve的基本功能來簡化軟體開發各週期中的日常事務。但enve不只是如此，enve被設計能夠整合入更大的系統中來簡化大型系統的設計。如果你扮演的是以下角色，更推薦你仔細研究enve能為你的組織帶來什麼助益：

  • 作為團隊經理人，用enve提供一套團隊標準化的開發流程，以容易上手的工具來縮短對成員的培訓。
  • 作為社群開發者，用enve來擺脫中央集權的控制，沒有任何人能阻礙你與大家分享成果。
  • 作為語言發明家，用enve當現成套件管理處理繁雜事務，讓你能將語言與框架設計聚焦在特定領域。
  • 作為系統服務商，用enve能夠脫離專用平台與專用協定的約束，無需任何花費即可整合入自家系統。

GETTING STARTED

由於enve不需要編譯，也不需要在系統中建立任何的永久檔案結構就能夠執行。這讓enve不需要安裝即可執行。就算要安裝，安裝方式將會更彈性而且跨平台。

如果你的系統上沒有Git，我們提供一個簡易的腳本來讓你可以一次性使用enve。

curl http://enve.gamelab.tw/script/bootenve.sh | sh enve help

否則你可以從[我們的源碼庫][]用git(1)克隆enve的原始碼並且可直接使用

git clone https://github.com/ranlempow/enve
./enve/bin/enve help

雖然不是必要的，但enve可以安裝自己到系統上，這樣之後可當作系統命令來執行

./enve/bin/enve install --host enve
enve help

或是你可以把enve嵌入你正在開發的原始碼中，這樣其他人有了你的原始碼之後不需先下載也能執行enve。

./enve/bin/enve init --embed <project_dir>
<project_dir>/.enve/bin/enve enter <project_dir>

我們建議先參考文件中的enveconcept(7)來了解enve的基本概念。
